Архитектура проекта
Проект построен на клиентской стороне (frontend-only), без серверной части. Это простая статическая веб-страница, которая может быть развернута на любом статическом хостинге (например, GitHub Pages, Vercel или локально).
Структура файлов

index.html: Основной HTML-файл. Содержит разметку экранов (онбординг, главный экран, карточка приложения, экран категорий). Включает шапку (поиск, категории, профиль), баннеры, сетки приложений, нижнюю навигацию.
style.css: Стилизация. Использует шрифт VK Sans, глобальные стили, анимации (для онбординга, галереи скриншотов). Адаптивная верстка с media queries для экранов ~430x932 px (мобильные устройства). Включает стили для сплеш-экрана, карточек, галереи, модального окна.
app.js: Логика на JavaScript. Загружает данные из data.json, рендерит экраны, обрабатывает события (клики, поиск, сортировка, галерея). Использует LocalStorage для хранения истории просмотров.
data.json: Данные о приложениях (id, name, icon, rating, category, developer, age, description, screenshots, banner, price, date, downloads). Хранится локально.
images/: Директория с иконками, баннерами, скриншотами, логотипом (svg/png).
fonts/: Шрифты VK Sans (Regular, Medium, Bold).

Технологии

HTML5: Разметка экранов и компонентов.
CSS3: Стили, анимации (transitions), flexbox/grid для布局, media queries для адаптивности.
JavaScript (ES6+): Логика, DOM-манипуляции, асинхронная загрузка данных (fetch), события (click, touch, input).
LocalStorage: Хранение истории просмотров и счетчиков.
Без фреймворков: Чистый vanilla JS для простоты и легкости.

Архитектура монолитная: все в одном файле JS, с глобальными переменными (apps, currentCategory, history, views). Функции разделены по задачам (загрузка данных, рендер, навигация, галерея).
Как что работает
Проект имитирует мобильное приложение в браузере. Логика построена вокруг переключения экранов (display: none/flex), рендеринга данных и обработки пользовательских взаимодействий.
Ключевые компоненты и их работа

Онбординг (Сплеш-экран):
Показывается при первом запуске (проверяется флаг в LocalStorage, но в коде — всегда при загрузке для demo).
Анимация: Логотип перемещается вверх, появляются текст и кнопка "Начать".
Кнопка "Начать" активируется после анимации, ведет на главный экран.
В app.js: Функция runSplash() управляет анимациями и переходом.

Главный экран (Витрина приложений):
Шапка: Поиск (фильтр по имени/разработчику), кнопка "Категории" (переход к экрану категорий или возврат), профиль (mock-алерт).
Баннеры: Карусель с автопрокруткой (5 сек), стрелками и точками. Рендерит приложения с banner != null. В app.js: initBannersCarousel(), goToSlide(), changeSlide().
Популярные категории: Топ-3 категории по количеству приложений, рендерятся как плитки. Клик — фильтрация и возврат на главный экран. В app.js: getTopCategories(), renderPopularCategories().
Сетки 2x2: Популярные (по downloads), Новые (по date), Бесплатные/Платные (по price и rating). Рендер в renderGridApps().
Поиск и сортировка: Фильтрует/сортирует список (по rating/new/name). В app.js: filterAndRender().
Нижняя навигация: Mock-кнопки (Интересное, Киоск, Моё) — не функциональны.

Карточка приложения:
Открывается по клику на карточку. Показывает детали (иконка, имя, разработчик, категория, рейтинг, возраст, описание, скриншоты).
Галерея скриншотов: Горизонтальная карусель с лупом (бесконечный цикл, клоны первого/последнего), стрелками, точками, свайпом (touch). Клик на скрин — модальное увеличение. В app.js: updateGallery(), changeGallerySlide(), openScreenshotModal().
Описание: Свёрнутое (4 строки) с кнопкой "Ещё"/"Свернуть". В app.js: Обработчик toggleBtn.onclick.
Похожие приложения: Фильтр по категории, рендер как список. В app.js: showAppDetail().
История: Добавляет в LocalStorage (history/views), используется для рекомендаций (но не реализовано полностью).

Экран категорий:
Плитки категорий (уникальные из data.json). Клик — фильтрация, переход на главный экран с отфильтрованным списком.
В app.js: renderCategories(), клики на плитки устанавливают currentCategory и вызывают filterAndRender().

Общая логика:
Загрузка данных: Асинхронно через fetch в loadData(). Если ошибка — отображает сообщение.
Навигация: Функция showScreen(id) скрывает/показывает экраны по ID.
Фильтрация/рендер: filterAndRender() обновляет все секции на основе поиска/категории/сортировки.
Анимации: CSS transitions для fade-out, scale, translate.
Модальное окно: Для увеличенного скриншота, закрытие по клику/крестику.


Дополнительные особенности

Адаптивность: Для мобильных экранов (clamp, media queries).
Локализация: Всё на русском.
Оптимизации: Lazy-loading изображений, скелетоны (заглушки) при рендере.

Как запустить проект

Локально:
Скачайте все файлы в одну директорию (index.html, style.css, app.js, data.json, images/, fonts/).
Откройте index.html в браузере (Chrome/Firefox, предпочтительно в мобильном режиме: F12 > Toggle device toolbar > iPhone 12/13).
